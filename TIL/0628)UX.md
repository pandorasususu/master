1. **제이콥의 법칙**

```
사용자는 자신에게 익숙한 제품을 통해 구축한 기대치를 그와 비슷해 보이는 다른 제품에도 투영
기존의 mental model을 활용하면 사용자가 새 모델을 익히지 않아도 바로 작업에 돌입할 수 있는 뛰어난 사용자 경험이 완성
변화 적용시 사용자에게 익숙한 모델을 한시적으로 이용할 권한을 부여해 불협화음 최소화
```

익숙한 인터페이스는 사용자가 목표를 쉽게 이루도록 도와줌
일반적인 디자인 관습을 따르면 사용자는 사이트의 콘텐츠/메세지/제품에 더욱 집중할 수 있음

멘탈 모델
=> 인간이 어떤 시스템과 그 작동 방식에 관해 알고 있다고 생각하는 바
=> 과거의 경험으로 쌓은 지식을 새로운 대상과 인터렉션할 때 사용

사례1: 스냅챗
=> 점진적인 변화 대신 **단기간의 극적 변화**
=> 급격한 변화로 인해 사용자 다수 경쟁 서비스로 이동, Mental Model Discordance

사례2: 구글
=> 2017년 유튜브 디자인 새 버전 공개했지만 강요하지 않고 서서히 익숙해지게 함
=> 사용자에게 새 디자인 **미리 확인**할 수 있게 함, 그리고 이 과정에서 **피드백** 보내거나 **이전 디자인**으로 복귀 가능

사례3: 벤츠
=> 2020 메르세데스 벤츠 EQC400의 좌석 제어부 버튼은 실제 좌석과 비슷한 모양으로 만들어짐
=> 어떤 버튼을 누르면 좌석의 어떤 부분이 작동하는 지 직관적으로 알 수 있음



2. **피츠의 법칙**

```
터치 요소는 적당한 "크기"로 제시되어야 하며,
목표 요소와 다른 "요소들 간의 거리"를 조절해 실수로 터치하지 않도록 하며,
모바일의 경우에는 유저가 "화면의 중앙"에 시선을 두는 것을 고려해야 함
```

사례1: 폼 텍스트 레이블
=> 레이블 요소와 입력 폼을 연결해 터치 대상 영역을 확장할 수 있다.
=> 이는 터치 정확도를 향상시키고, 사용자 경험을 증진

사례2: 폼 전송 버튼
=> 맨 마지막에 입력하는 폼 주변에 배치 전송 버튼 배치

사례3: 애플의 아이폰6, 아이폰6 플러스 모델
=> 화면 상단 간편 접근: 화면 상단에 놓인 항목을 화면 하단에 내려 한손 사용자 경험 향상



3. **힉의 법칙**

```
의사결정에 걸리는 시간은 선택지의 개수와 복잡성과 비례해 증가
인지 부하를 줄이려면 복잡한 작업을 잘게 나눠야
추천 선택지를 강조해서 사용자의 부담 줄이기
신규 사용자의 인지 부하를 줄이려면 온보딩onboarding을 점진적으로 진행
추상적이라고 느껴질 정도로 단순화하지 않도록 주의
```

사용자가 부담을 느끼지 않도록 정보를 종합하고 정리해서 제시해야 
=> 이를 위해서는 사용자의 목표를 정확히 이해하고 이를 방해하는 요소를 줄이거나 없애야됨
=> **인지 부하**: 인터페이스를 익히고 인터렉션하는 데 필요한 정신적 자원의 양

사례1: TV 리모컨
=> 버튼이 너무 많아져 사용자 경험이 저하, 이를 개선하기 위해 최소 버튼만 넣은 리모컨 등장

사례2: 구글 검색결과
=> 전체, 동영상, 이미지, 뉴스 등 유형에 따라 필터링 기능 제공

사례3: 슬랙 
최초 사용자의 온보딩 시, 슬랙봇의 메시징 기능을 제외한 모든 기능을 숨겨 사용자 부담 줄여줌

지나친 단순화
아이콘이 지니는 의미는 사람마다 다르게 받아들임, 이를 설명하는 **텍스트를 삽입**하는 게 권장됨



4. **밀러의 법칙**

```
밀러 연구의 핵심
=> 단기 기억에는 5개~9개 항목만 저장된다 (x)
=> 단기 기억에는 한계가 있으며, 정보를 유의미한 덩어리로 정리하면 이를 최적화할 수 있음

사용자가 쉽게 처리하고 이해하고 기억할 수 있게 콘텐츠 덩어리를 작게 나눠 정리
단기 기억 용량은 사람에 따라, 그리고 기존 지식과 상황적 맥락에 따라 달라짐
```

기억 범위에 더 큰 영향을 미치는 건 정보의 기본 단위인 bit의 양이 아니라 **정보 덩어리chunk**의 개수
=> 덩어리: 인지심리학에서 사용하는, 한 그룹으로 묶여 기억에 저장되는 기본 단위

chunk로 나누어 접근(덩어리화)
=> 정보 밀도가 높은 경험을 덩어리화하면 콘텐츠 구조가 형성됨 ex) 뉴스 헤드라인
=> 중요한 정보를 쉽고 빠르게 구분할 수 있게 해줌



5. **포스텔의 법칙**

```
자신이 행하는 일은 엄격하게
=> 서비스 제공 시 안정성과 접근성을 보장해야 함 
=> 기기의 스펙에 상관 없이 모두에게 정상 작동하는 인터페이스 제공해야

남의 것을 받아들일 때는 너그럽게
=> 사용자의 입력은 포맷이나 입력 메커니즘을 막론하고 무조건 수용해야 함

다양한 가능성에 대해 잘 예측하고 대비할 수록 디자인 회복탄력성 개선
사용자의 가변적인 입력을 수용해서 기계가 이해할 수 있는 방식으로 해석하라. 입력의 한계를 정의하고 사용자에게 명확한 피드백을 제공하라
```

사례1: HTML, CSS
제작상의 실수나 브라우저에서 지원하지 않는 기능 등의 문제를 에러를 발생시키지 않고 넘어감

사례2: 입력 폼
이메일 주소나 비밀번호처럼 이미 확보한 정보는 중복으로 요청하지 않고, 필요한 것만 묻는 방식으로 폼을 채우는 데 드는 노력을 최소화 해야(의사결정 피로감 관리)

사례3: 반응형 디자인
유동형 그리드, 가변 이미지, 미디어 쿼리를 활용해 다양한 시청 맥락에 맞게 웹사이트의 콘텐츠가 유동적으로 반응

디자인 **회복탄력성**
다양한 상황에 대해 예측하고 대비해  사용자의 부담을 줄여 사용자경험을 보장해야 함



6. **피크엔드 법칙**

```
인간은 경험 전체의 평균이나 합계가 아니라, 절정의 순간과 마지막 순간에 느낀 감정을 바탕으로 경험을 판단
=> 사용자 여정 중 가장 강렬한 순간과 마지막 순간에 집중
사람들은 긍정적인 순간보다 부정적인 순간을 더 생생하게 기억
```

인지편향
상황을 철저하게 분석하기보다 효율적으로 신속한 결정을 내리게 해주는 일종의 정신적 지름길
인지편향의 한 종류인 피크엔드 규칙은 기억 회상을 방해하는 기억 편향이라고도 알려짐
=> 인간은 자신의 감정을 강하게 자극한 사건을 그렇지 않은 사건보다 더 잘 기억, 이는 우리가 경험을 인지하는 방식에도 영향을 미침

사례1: 우버
비즈니스 모델 특성상 대기시간이 불가피하다는 것을 인지
=> 유휴 시간에 대한 거부감(도착할 때까지 차량 이동경로를 애니메이션으로 표현), 
작동상의 투명성(도착 예상 시간이 계산되는 방법 공개), 
점진적 목표 달성 효과(고객에게 차량 탑승이라는 목표 꾸준히 상기)
에 주목해 불편을 감소시키고자 노력
=> 요청 취소율을 낮추고 서비스 이용 중 부정적 감정 일정 수준으로 관리

여정지도
최종 사용자가 경험 도중에 감정이 언제 절정에 이를지 알아내는 도구
사용자가 임무나 목표를 성취하는 과정을 서술함으로써 사용자가 제품 혹은 서비스를 사용하는 방법을 시각화할 때 매우 유용
렌즈: 사용자의 페르소나와 사용 시나리오 제시
경험: 타임라인을 따라가며 최종 사용자의 행동, 사고방식, 감정을 조명
통찰: 경험에서 드러난 교훈, 경험 개선을 위한 지표 목록이나 그 지표를 담당할 부서 정보



7. **심미적 사용성 효과**aesthetic-usability effect

```
보기 좋은 디자인은 사용자로 하여금 제품이나 서비스의 사용성이 뛰어나다는 생각이 들게하며,사소한 사용성 문제에 비교적 관대해짐
단, 시각적으로 만족스러운 디자인은  사용성 테스트 중에 문제가 드러나는 것을 방해할 수 있음을 경계
```

자동 인지 처리automatic cognitive process
진행 중인 작업과 관련된 정보와 관련없는 정보를 빠르게 구분하고 관련없는 정보는 무시
=> 웹사이트에 대한 인상은 1/20초만에 결정되고, 머무는 시간이 증가해도 인상은 바뀌지 않음

독일의 전자회사 브라운Braun 수석 디자이너 디터 람스Dieter Rams
less but better: 형태가 기능을 따를 것을 강조(=모든 **디자인 디테일은 기능적 목적이 있어야** 함)
기능적 미니멀리즘과 심미성이 조화를 이룸

사용자 테스트 진행할 때 좋은 디자인이 사용성 문제를 가리는 것을 경계해야 함



8. **폰 레스토프 효과**

```
비슷한 사물이 여러 개 있으면 그중에서 가장 차이가 나는 한 가지만 기억할 가능성이 크다.
중요한 정보나 핵심 동작은 시각적으로 눈에 띄게 하되, 시각적 요소를 남발해 강조할 항목이 눈에 띄지 않는 상황은 지양
움직임을 활용해서 대비를 전달할 때는 움직임에 민감한 사용자를 주의 깊게 고려
```

선택적 주의력selective attention
뇌의 용량과 지속 시간 면에서 제한이 있기 때문에 관련 없는 정보를 무시하고 관련 있는 정보에 집중/주의
**배너 무시**banner blindness: 사용자가 광고라고 인식한 요소를 무시하는 경향
광고 같이 불필요한 것은 무시하되, 네비게이션 메뉴, 검색 상자, 헤드라인 등 도움이 되는 요소에 집중
=> 광고 주위의 중요한 요소 배치하면 주의 분산

사례1: 확인 대화상자
삭제 행위는 대체로 위험 
=> 따라서 헤더에 경고 아이콘 삽입, 삭제 버튼색을 빨간색으로 지정 등의 추가 요소가 효과적

사례2: 구글 머티리얼 디자인
구글은 Floating Action Button(FAB)의 디자인, 위치, 동작에 대해 가이드라인 제시
=> 사용자에게 익숙한 패턴 제공해 경험 개선(제이콥 법칙과 유사)

사례3: 기타
앱의 알림 배지 / 뉴스의 헤드라인 등

레스토프 효과 적용 시 주의사항
=> 시각적 차별화를 남용하지 말고 명확한 목적을 가지고 적용
=> 사용자의 상황을 고려해 포괄적인 접근성 보장해야(시각장애, 신경장애 등)



9. **테슬러의 법칙**

```
모든 프로세스에는 디자인 시 처리할 수 없는 기본적인 복잡성이 존재하므로, 시스템이나 사용자 중 한쪽이 감당해야 함
내재된 복잡성을 디자인과 개발 과정에서 처리하면 사용자의 부담을 최소로 줄일 수 있다.
추상적으로 느껴질 정도로 인터페이스를 단순화해서는 안된다.
```

사례1: 이메일
이메일을 보내기 위해서는 수신자와 발신자에 대한 정보가 필수
=> 로그인 사용자의 이메일 주소를 발신자에 기입, 사용자가 수신자 정보 기입하는 형태로 사용자 수고 줄임
=> 지메일은 스마트 편지쓰기 기능을 통해 메일 내용 스캔 후 적절한 단어/구절을 제안해 타이핑 수고를 덜어줌

사례2: 이커머스
배달에 필요한 정보가 로그인 유저의 정보를 받아 자동으로 완성됨



10. **도허티 임계**

```
사용자의 주의가 분산되는 것을 막는 동시에 생산성도 향상시키려면 시스템 피드백을 0.4초 이내에 제공하라
반응 시간을 개선하고 체감 대기 시간을 줄이려면 체감 성능을 활용하라
애니메이션은 로딩이나 프로세싱이 진행되는 동안 사람들의 시선을 끄는 한가지 방법이다.
설사 정확하지 않다고 해도 진행표시줄을 보여주면 사용자는 대기 시간에 좀 더 관대해진다
실제 작업이 훨씬 빨리 완료되더라도, 의도적으로 작업 완료를 늦게 알리면 체감 가치를 높이고 신뢰를 형성하는 데 도움이 되기도 한다. 
```

웹/모바일 페이지 용량은 꾸준히 증가
2010년~2011년 pc: 608kb, mobile: 261kb 
=> 2019 pc: 2mb, mobile: 1.7mb

도허티 임계: 컴퓨터 반응 시간이 생산성에 불균형한 영향을 미침
페이지 처리 시간을 도허티 임계 이하로 줄일 수 없을 때 다음과 같은 기법을 적용할 수 있음

사례1: skeleton screen 
페이스북은 콘텐츠 처리 하면서 skeleton screen 적용
=> 콘텐츠가 로딩되는 동안 콘텐츠 영역에 임시로 placeholder 블록을 표시
=> 사용자는 속도와 반응성이 실제보다 더 낫다고 인지 

사레2: blur up
큰 이미지를 표시할 때 작은 크기로 로딩하고 점차 확대해 표시
=> 저해상도 이미지가 커지면서 노이즈가 생길 때 가우시안 블러 활용해 감춤

사례3: progress indicator
요청한 작업이 처리되고 있다는 사실을 분명히 알 수 있음
사용자가 기다리는 동안 볼거리 제공해 대기 프로세스에서 오는 불만 완화

**반응이 너무 빠를 때도 문제가 발생**
=>사용자가 변화를 알아차리지 못함 / 변화에 대해 이해하기 어려움 / 작업에 대한 체감 가치, 신뢰도가 감소 



11. **힘에는 책임이 따른다**

기술이 행동을 형성하기 위해 활용하는 기법
=> 간헐적이고 불규칙한 보상, 사회적 확증, 호혜성
=> 무한 루프: 사용자가 사이트에 머무는 시간 증가
=> 기본 설정: 비록 마음에 들지 않더라도 기본 설정을 고수 
=> 방해 요소 제거: 사용자가 특정 행동하도록 그 과정 전부를 생략
=> 다크 패턴: 사용자 참여도 증진 위해 불필요한 정보 공유, 마케팅 정보 수신 허용 등 ex) 의류 재고 개수
